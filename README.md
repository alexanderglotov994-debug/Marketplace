# Дашборд кейсов доработок

Статическое HTML-приложение для GitHub Pages в стилистике Mango Office / Mango Academy. Источник данных хранится рядом с сайтом, без сервера и базы данных.

## Что уже есть

- `index.html` — одностраничное приложение
- `admin.html` — внутренняя админ-страница для обновления базы
- `styles.css` — адаптивные стили
- `app.js` — загрузка данных, фильтры, поиск и выдача карточек
- `data/cases.json` — тестовый датасет
- `data/cases-template.csv` — Excel-совместимый шаблон для обновления базы
- `assets/mango-academy-logo.png` — логотип в проекте
- `scripts/export-cases.mjs` — экспорт CSV в JSON
- `package.json` — команды для генерации JSON

## Как это работает

Сайт читает файл `data/cases.json`. Для GitHub Pages это самый надёжный вариант: страница остаётся полностью статической, а данные можно менять отдельно.

## Шаблон данных

Файл `data/cases-template.csv` можно открыть в Excel.

- Каждая строка — один кейс.
- Поля `Отрасли`, `Продукты`, `Подразделения` поддерживают несколько значений.
- Несколько значений в одной ячейке разделяются символом `|`.

## Сценарий обновления Excel -> JSON

1. Открыть `data/cases-template.csv` в Excel.
2. Обновить строки кейсов.
3. Сохранить файл как `CSV UTF-8` с тем же именем.
4. Выполнить команду:

```bash
node scripts/export-cases.mjs
```

Или:

```bash
npm run export:data
```

После этого будет обновлён `data/cases.json`, который читает сайт.

Пользовательская страница `index.html` не содержит служебных блоков про загрузку данных. Вся служебная информация вынесена в `admin.html`.

## Формат JSON

```json
{
  "id": "case-001",
  "title": "Название доработки",
  "description": "Краткое описание",
  "industries": ["Ритейл", "E-commerce"],
  "products": ["CRM", "B2B-портал"],
  "divisions": ["Продажи", "Коммерческий отдел"],
  "url": "https://example.com/cases/case-001"
}
```

## Примечание по Excel

CSV выбран как основной шаблон, потому что он надёжно работает в GitHub-репозитории и легко редактируется в Excel. Если понадобится, позже можно добавить отдельный импорт из `.xlsx`, но для статического GitHub Pages сценарий `CSV -> JSON` проще и стабильнее.
