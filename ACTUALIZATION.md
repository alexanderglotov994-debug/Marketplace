# Актуализация кейсов

Сайт читает `data/cases.csv` напрямую. Экспорт в JSON больше не нужен.

## Самый простой вариант

1. Открыть `data/cases.csv` в GitHub через веб-интерфейс.
2. Нажать `Edit this file`.
3. Добавить, удалить или поправить строки кейсов.
4. Закоммитить изменения.
5. Дождаться публикации GitHub Pages.

## Вариант через Excel

1. Открыть `data/cases.csv` в Excel.
2. Обновить строки кейсов.
3. Если в поле несколько отраслей, продуктов или подразделений, разделять значения через `|`.
4. Сохранить файл как `CSV UTF-8`.
5. Загрузить обновлённый файл в репозиторий и закоммитить.

## Что важно

- Каждая строка — отдельный кейс.
- Обязательные колонки: `Название доработки`, `Описание`, `Отрасли`, `Продукты`, `Подразделения`, `Ссылка`.
- После коммита GitHub Pages публикует изменения автоматически.
- По GitHub Docs публикация обычно занимает до 10 минут.
